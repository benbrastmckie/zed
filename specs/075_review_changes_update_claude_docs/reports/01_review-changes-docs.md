**Key architectural changes**:
1. CLAUDE.md is now a **computed artifact** regenerated by `generate_claudemd()` on every load/unload, not maintained via section injection
2. Section markers (`<!-- SECTION: extension_X -->` / `<!-- END_SECTION: extension_X -->`) are vestigial and have been removed from CLAUDE.md
3. `inject_section()` and `remove_section()` still exist in merge.lua but are not called for CLAUDE.md management
4. Core system files are now managed as a "core" extension rather than being separate from the extension system
5. Unloading a dependency extension is now a **hard block** (returns false immediately) instead of a confirmable warning
6. Conflict detection now shows a **confirmation dialog** instead of unconditionally aborting
