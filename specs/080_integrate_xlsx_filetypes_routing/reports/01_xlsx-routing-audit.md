## Findings

### Item-by-Item Audit

#### (1) extensions.json -- add skill-xlsx as installed_dirs entry and add context paths

**Status: NOT DONE**

The `filetypes` entry in `extensions.json` does not include:
- `skill-xlsx` in `installed_dirs` (currently lists: skill-filetypes, skill-filetypes-spreadsheet, skill-presentation, skill-scrape, skill-docx-edit, project/filetypes)
- `xlsx-agent.md` in `installed_files`
- `xlsx.md` in `installed_files`
- `skill-xlsx/SKILL.md` in `installed_files`
- xlsx-related paths in `merged_sections.index.paths`

**However**: This file is auto-generated by the extension loader. Manually editing it would be overwritten on next sync. The correct fix is to ensure the source extension has the files, then re-run the loader.

#### (2) skill-filetypes/SKILL.md -- add xlsx as a routed format

**Status: PARTIALLY DONE (indirect)**

The skill-filetypes SKILL.md does mention `.xlsx` in its file extension detection list (line 44), but it routes xlsx through the filetypes-router-agent to the spreadsheet-agent for table conversion. It does NOT route xlsx to skill-xlsx for create/edit/analyze operations.

The "When NOT to trigger" section (line 60) says "Spreadsheet-specific operations (use skill-spreadsheet)" but does not mention skill-xlsx for xlsx creation/editing/analysis.

**What's needed**: The skill-filetypes SKILL.md should clarify that xlsx **creation/editing/analysis** routes to skill-xlsx, while xlsx **table extraction** routes to skill-filetypes-spreadsheet.

#### (3) convert.md -- add xlsx operation mode

**Status: ALREADY DONE (was there before task 79)**

The convert.md already has xlsx in the supported conversions table (line 58: XLSX to Markdown). The `/convert` command handles xlsx-to-markdown conversion via markitdown. This is the correct behavior -- `/convert` extracts content, while `/xlsx` creates/edits/analyzes.

#### (4) CLAUDE.md -- add skill-xlsx to filetypes extension section

**Status: NOT DONE in installed copy**

The zed project's `.claude/CLAUDE.md` Filetypes Extension section (lines 409-431) does NOT mention:
- skill-xlsx / xlsx-agent in the Skill-Agent Mapping table
- /xlsx command in the Commands table

The **source** EXTENSION.md WAS updated by task 79 and includes both. But the installed CLAUDE.md was not regenerated from the updated source.

#### (5) Create /xlsx command

**Status: DONE (by task 79)**

Task 79 created `.claude/commands/xlsx.md` directly in the zed project. The file is complete and functional (264 lines). It delegates to skill-xlsx, which spawns xlsx-agent.

### Source Extension Gap (Critical Finding)

Task 79 updated the source extension's **metadata files** (manifest.json, EXTENSION.md, index-entries.json) to reference xlsx, but did NOT create the actual files in the source extension directory:

**Missing from `~/.config/nvim/.claude/extensions/filetypes/`:**
- `skills/skill-xlsx/SKILL.md` -- does not exist
- `agents/xlsx-agent.md` -- does not exist
- `commands/xlsx.md` -- does not exist

**Present only in zed project `~/.config/zed/.claude/`:**
- `skills/skill-xlsx/SKILL.md` -- exists (created by task 79)
- `agents/xlsx-agent.md` -- exists (created by task 79)
- `commands/xlsx.md` -- exists (created by task 79)

This means:
1. Re-syncing the extension would **lose** the xlsx files (they are not in the source)
2. Other projects loading the filetypes extension would NOT get xlsx support
3. The loader never registered these files in extensions.json because it never installed them from the source

### Routing Architecture

Current routing for xlsx files:

| Entry Point | Route | Destination | Purpose |
|-------------|-------|-------------|---------|
| `/convert file.xlsx` | skill-filetypes -> filetypes-router-agent -> spreadsheet-agent | Extract to markdown | Content extraction |
| `/table file.xlsx` | skill-filetypes-spreadsheet -> filetypes-spreadsheet-agent | LaTeX/Typst table | Table conversion |
| `/xlsx file.xlsx "instruction"` | skill-xlsx -> xlsx-agent | Create/edit/analyze | Full spreadsheet ops |
| `/edit file.xlsx "instruction"` | skill-xlsx -> xlsx-agent | Edit existing xlsx | Office edit routing |

The /xlsx and /edit xlsx routes work because the files exist in the zed project. But they are not "properly installed" through the extension system.

## Decisions

1. The proper fix is to copy the three files from the zed project into the source extension directory, then re-run the extension loader to properly register them
2. The CLAUDE.md update will happen automatically when the extension loader regenerates it from the updated EXTENSION.md (which already includes xlsx)
3. The extensions.json update will happen automatically when the loader installs from the source
4. The context/index.json update will happen automatically when the loader merges index-entries.json
