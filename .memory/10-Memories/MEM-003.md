Don't create task directories at creation time; only create them when artifacts are actually written.

**Why:** Creating empty directories at task creation adds filesystem noise and makes it harder to tell which tasks have real work products. The agent system should defer directory creation until a report, plan, or summary needs to be stored.

**How to apply:** When `/task` creates a new task, update only TODO.md and state.json. Do not create `specs/{NNN}_{SLUG}/` until `/research`, `/plan`, or `/implement` produces an artifact that needs to be written there.
