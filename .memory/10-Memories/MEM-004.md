Tasks need actual report files to be marked RESEARCHED.

**Why:** Marking a task as RESEARCHED without a corresponding report file in `specs/{NNN}_{SLUG}/reports/` creates a false sense of progress. Downstream commands like `/plan` expect research artifacts to exist when the status says research is complete.

**How to apply:** Only transition a task to [RESEARCHED] status after a research report has been written to the task's reports directory. If the research skill completes without producing an artifact, the task should remain in [RESEARCHING] or revert to [NOT STARTED].
