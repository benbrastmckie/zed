## Decision

Disabled Claude Code's auto-memory system globally by setting `"autoMemoryEnabled": false` in `~/.dotfiles/config/claude-settings.json` (deployed via Home Manager to `~/.claude/settings.json`).

## Rationale

- Auto-memory files in `~/.claude/projects/` are opaque, not version-controlled, and not portable across machines.
- The per-repo `.memory/` vault provides a structured, version-controlled alternative with frontmatter, indexing, and the `/learn` command for explicit memory creation.
- All 4 existing auto-memory entries (2 files + 2 index-only) were migrated to `.memory/10-Memories/MEM-001.md` through `MEM-004.md`.

## Rollback

To re-enable auto-memory:
1. Set `"autoMemoryEnabled": true` in `~/.dotfiles/config/claude-settings.json`
2. Run `home-manager switch`
3. The `.memory/` entries remain valid regardless (no conflict).
