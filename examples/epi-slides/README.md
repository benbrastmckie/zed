# epi-slides -- Conference Talk: The /epi Workflow

A 14-slide Slidev deck walking a mixed clinical/informatics audience through
the `examples/epi-study/` synthetic RCT as a showcase of the `/epi` Claude
Code workflow.

> **Synthetic data throughout.** Every number in this deck is generated by the
> data-generating process shown on slide 7. None of the findings are clinical
> evidence.

## Overview

- **Talk mode**: CONFERENCE (18 min + 2 min Q&A)
- **Slide count**: 14
- **Theme**: `academic-clean` (local Slidev theme under `theme/`)
- **Source of truth**: `../../specs/029_talk_epi_study_walkthrough/reports/02_talk-research.md`
- **Upstream data/results**: `../epi-study/` (read-only; never modified by the deck)

## Prerequisites

- **Node.js** >= 18 (tested with 24.14)
- **pnpm** >= 9 (tested with 9.15.9)
- **Playwright Chromium** -- required for `pnpm export` (PDF).
  - On NixOS, the Playwright-bundled chromium fails with
    `libglib-2.0.so.0: cannot open shared object file`. Work around by
    installing chromium via nix profile and pointing Playwright at it:
    ```bash
    nix profile install nixpkgs#chromium
    export PLAYWRIGHT_CHROMIUM_EXECUTABLE_PATH="$(which chromium)"
    ```
  - On Debian/Ubuntu, `apt install libglib2.0-0` is usually enough.

## Commands

```bash
pnpm install             # install dependencies
pnpm dev                 # live preview at http://localhost:3030
pnpm build               # SPA build -> dist/
pnpm export              # PDF export -> dist/slides-export.pdf (14 pages)
pnpm export-png          # per-slide PNGs -> dist/slides-png/
pnpm sync-assets         # re-copy task-28 outputs into public/assets/
```

## Output Layout

```
dist/
├── index.html                   # SPA entry
├── 404.html                     # Slidev 404
├── _redirects                   # Netlify-style redirects
├── assets/                      # JS/CSS/font bundles (slide chunks per page)
└── slides-export.pdf            # (generated by `pnpm export`, not checked in)
```

## How to Update

If `examples/epi-study/` is ever re-run (e.g. another round of enrichment),
re-sync the assets and rebuild:

```bash
bash scripts/sync-assets.sh    # idempotent rsync from ../epi-study/
pnpm build
pnpm export                    # optional; requires working chromium
```

The sync script pulls tables, receipts, and the Quarto CONSORT render from
their canonical locations in `examples/epi-study/`. Never edit files under
`public/assets/` by hand -- the deck should always trace to a source of truth
in `../epi-study/`.
